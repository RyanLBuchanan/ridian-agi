# Frontend UX Notes

## Intent

The Little Ridian AGI frontend should feel like a calm workspace instrument, not a generic chatbot.

The desired feeling is:

- warm
- intelligent
- restrained
- lightly futuristic
- premium
- trustworthy

## UX Pattern

The interface is organized as a three-part workspace shell:

1. Left navigation rail

- Persistent orientation.
- Highlights core areas without overwhelming the user.
- Reads more like a systems panel than an app drawer.

2. Central command surface

- Primary focus area.
- Holds conversation, command input, and latest run metadata.
- Should feel clear, calm, and legible over long work sessions.

3. Right context rail

- Secondary operational context.
- Holds memory, plans, tools, and trace/activity placeholders.
- Designed for progressive disclosure rather than constant noise.

## Visual Language

- Warm neutrals rather than cold grayscale or neon-heavy sci-fi.
- Soft glassy surfaces with restrained depth.
- Strong hierarchy through spacing, typography, and subtle contrast.
- Pills and metadata chips should feel operational, not decorative.

## Interaction Posture

- The conversation remains central.
- Operational metadata is visible but secondary.
- The interface should suggest disciplined orchestration and governance.
- Tooling, memory, and trace surfaces should look trustworthy before they look powerful.

## Placeholder State Principles

- Placeholder panels should still teach the structure of the system.
- Empty states should describe what the area is for.
- Metadata areas should remain stable even when some backend fields are absent.

## Later Enhancements

- Per-message execution metadata.
- Rich trace timeline with expandable steps.
- Live tool decision states.
- Real memory cards and retrieval matches.
- Responsive panel collapse behavior for narrower screens.
